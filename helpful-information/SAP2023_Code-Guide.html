Inventory

Attach this script to the Player. It's a singleton, so there can only be one.

Properties of Inventory:

  • capacity (int): the maximum amount of items that can be held in the inventory.
  • order (list of string): a list of the names of the items in the inventory, in order.


To use functions from Inventory, type: Inventory.Instance._function_

Example:

  • Inventory.Instance.isFull();


Relevant functions:

public void AddItem(string something)

This function is called by Pickup. It takes the name of an item and adds it into an empty slot. If there are no empty slots, nothing happens.

public void RemoveItem(string something)

This function is called by Unlockable after something has been used successfully. It takes the name of and item and removes it from the inventory.

public bool isFull()

This function might be useful if you want to check if the inventory is full. There's also isEmpty() if you want to check if the inventory is empty.

Door

This is a type of Item.Unlockable.

Properties of Door:

  • locked (bool): whether the door is locked or not.
  • nextRoom (string): the name of the room (scene) the door connects to.


Relevant functions:

public void unlock()

This function should be called when the Player is within range and presses the "interact button". It checks if the Player's inventory has its "key," then updates accordingly.

If the Player has the correct "key," the door should open (Unlockable.locked = false; ).

After being unlocked, the Player can then use the door to go to other rooms.

Otherwise, nothing will happen.

Container

This is a type of Item.Unlockable.


Relevant functions:

public void unlock()

This function should be called when the Player is within range and presses the "interact button". It checks if the Player's inventory has its "key," then updates accordingly.

If the Player has the correct "key," the container opens (Unlockable.locked = false; ).

After being unlocked, a new item will appear (whatever was stored in the container) that the Player can interact with.

Otherwise, nothing will happen.

Game Design

Creating Items

You can create items using the ItemData scriptable object. These scriptable objects should be saved in Assets > _Data > _itemData.

Properties of an ItemData:

  • itype (ItemType): the type of item. Possible options are:
    • MISC: the most basic item.
    • KEY: an item that unlocks another item.
    • DOOR: an item another item can unlock.
    • CONTAINER: an variante of a door, except it holds another item.
  • sprite (Image): the sprite for the item
  • comment (string): What the PC will say when they interact with the item.
  • canInteractWith (list of strings): The list of names of items that can interact with this item.
    • NOTE: This property should only be used for static items that the player doesn't pick up.

Make sure to add the new ItemData to the ItemDatabase (found in the same directory).

Next, create the actual game object. There are already basic prefabs for each type of item in Assets > Prefabs.

Every item object has a soundEffect variable. It's not mandatory, but the name of the sound effect the item will have can be put there.

For the item scripts, Pickup doesn't need any data, but Unlockable does. To make an Unlockable a "container," specify a game object for Unlockable.treasure.

Creating Rooms

You can create items using the Room prefab saved in Assets > Prefab.

The Room prefab already a rigidbody and composite collider for collision with the Player. Add the background sprite to the Room, update the wall and door placements to match, and add whatever items.

When the Room is done, put it in its own scene. Don't forget to put the name of the scene in whichever doors will lead the player to it.

Sound Design

AudioManager

The AudioManager keeps a list of all the audio (BGM & SFX) that out game will be using.

To add audio to the AudioManager's list, go to the prefab and add an entry to its list of Sound. Then, just fill in the information.

Properties of Sound:

  • name (string): the name of the sound. this will be used to play it.
  • clip (AudioClip): the actual audio file.
  • volume (float): the volume of the audio clip.
  • pitch (float): the pitch of the audio clip.
  • loop (bool): whether the audio clip will loop or not (set as true for BGM, false for most other things).
